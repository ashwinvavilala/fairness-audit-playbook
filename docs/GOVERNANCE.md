# Governance Model

## Roles

### Maintainers
- Own long-term direction of the playbook  
- Approve major changes  
- Ensure alignment with Responsible AI principles  

### Contributors
- Submit improvements  
- Report issues  
- Provide domain expertise  

## Decision-Making

- **Minor changes:** Maintainer approval  
- **Major changes:** Consensus among maintainers + Responsible AI team  
- **Documentation changes:** Must follow repository structure and style  

## Release Cadence

- **Quarterly minor releases**  
- **Annual major release**  
- **Patch releases** for urgent fixes
